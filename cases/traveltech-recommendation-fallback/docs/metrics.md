# Metrics

## Technical metrics

- `fallback_rate`: share of recommendation responses generated by fallback.
- `hybrid_rate`: share of hybrid responses.
- `ml_timeout_rate`: share of requests where ML exceeded 3 seconds.
- `ml_unavailable_rate`: share of failed ML requests.
- `low_confidence_rate`: share of ML responses below confidence threshold.
- `gateway_response_time_p50/p95/p99`: Recommendation Gateway response time.
- `fallback_response_time_p50/p95/p99`: fallback engine response time.

## Product metrics

- recommendation CTR;
- booking conversion from recommendation click;
- save rate;
- hide rate;
- complaint rate;
- post-trip feedback score;
- conversion comparison between ML, fallback and hybrid modes.

## Analytics questions

- How often does fallback protect the user flow?
- Does fallback have acceptable CTR compared with ML?
- Which fallback reason has the biggest business impact?
- Does hybrid mode outperform pure fallback?
- Are some destinations or seasons more exposed to low-confidence ML?
