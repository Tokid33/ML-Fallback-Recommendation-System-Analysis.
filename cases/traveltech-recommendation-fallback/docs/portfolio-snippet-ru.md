# Короткое описание для портфолио / резюме

TravelTech Recommendation Fallback Case: спроектировал резервный сценарий рекомендаций при отказе ML-сервиса, timeout больше 3 секунд и low confidence. Описал fallback engine на внутренних данных, режимы ML/fallback/hybrid, API-ответ, плавное UX-переключение, privacy-ограничения, логирование, метрики и acceptance criteria.

Кейс показывает, что ML-фича в продукте должна проектироваться как часть надежной системы, а не как магическая модель без резервного сценария.
