# Acceptance Criteria

## Критерии приёмки для TravelTech Recommendation Fallback Case

## Scenario 1: ML-сервис недоступен

**Given** ML-сервис возвращает ошибку 5xx или network error  
**When** пользователь открывает блок рекомендаций  
**Then** Recommendation Gateway включает fallback engine  
**And** frontend получает рекомендации в едином формате  
**And** пользователь не видит техническую ошибку  
**And** в логах фиксируется `reason = ml_unavailable`

## Scenario 2: ML-сервис отвечает дольше 3 секунд

**Given** ML-сервис не вернул ответ за 3 секунды  
**When** Recommendation Gateway ожидает результат  
**Then** система прекращает ожидание ML для текущего пользовательского ответа  
**And** возвращает fallback-рекомендации  
**And** в логах фиксируется `reason = ml_timeout`

## Scenario 3: ML вернул низкий confidence

**Given** ML-сервис вернул рекомендации с confidence ниже порога  
**When** Recommendation Gateway проверяет качество ответа  
**Then** система использует fallback или hybrid-режим  
**And** слабые рекомендации ML не показываются пользователю без замены  
**And** в логах фиксируется `reason = low_confidence`

## Scenario 4: ML ответил после показа fallback

**Given** fallback-рекомендации уже показаны пользователю  
**When** ML-сервис позже возвращает качественный ответ  
**Then** текущий список не должен резко перестраиваться  
**And** пользователь не теряет фокус в интерфейсе  
**And** система может показать мягкое предложение обновить подборку

## Scenario 5: Групповая поездка

**Given** поездка содержит нескольких участников  
**When** система строит fallback-рекомендации  
**Then** учитываются интересы участников  
**And** конфликтующие варианты понижаются в выдаче  
**And** приоритет получают компромиссные активности

## Scenario 6: Единый контракт для frontend

**Given** источник рекомендаций может быть ML, fallback или hybrid  
**When** frontend получает ответ от Recommendation Gateway  
**Then** структура ответа остаётся одинаковой  
**And** frontend не должен иметь отдельную бизнес-логику под каждый источник

## Scenario 7: Приватность

**Given** fallback использует популярность среди других пользователей  
**When** формируется объяснение рекомендации  
**Then** показывается только агрегированная анонимная информация  
**And** поведение конкретных пользователей не раскрывается
